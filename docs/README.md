# 文档总览

本目录用于后续开发协作,覆盖当前项目的核心约定与实现细节.

## 文档索引

- `docs/architecture.md`: 系统架构与数据流
- `docs/api.md`: 后端 HTTP + SSE 接口协议
- `docs/backend.md`: 后端模块说明与扩展方式
- `docs/frontend.md`: 前端页面结构与流式处理
- `docs/development.md`: 本地开发,测试与排障流程

## 阅读顺序建议

1. 先看 `architecture.md` 了解全局.
2. 再看 `api.md` 明确前后端契约.
3. 改后端看 `backend.md`,改前端看 `frontend.md`.
4. 提交前按 `development.md` 的检查清单执行.
